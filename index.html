Thinking Like a Programmer

  • Procedural Thinking
    • - Create perfectly clear and unambiguous instructions for a computer to follow. (Because computers are stupid)
  • Abstract Thinking
    • - Find the similarities between seemingly different things.
  • Systems Thinking
    • - Break a big problem down into smaller, more manageable components
  • Technological Empathy
    • - Understand how computers (& other tech) work/operate
  • Debugging
    • - Systemic process for identifying the cause of a computer program that doesn't work as intended in some fashion
    ----->Collect Evidence
    ----->Generate Theories
    ----->Test Theories
    ----->Resolve the Problem
